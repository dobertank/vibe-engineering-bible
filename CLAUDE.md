# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это за репозиторий

Это **документационный репо** — корпоративная политика по работе с AI-ассистентами для блока в 2000+ человек. Кода, тестов, сборки, линтеров здесь нет. Не предлагай команды `npm`, `pytest`, `make` и т.п. — их некуда применять.

Содержит два файла на русском языке:

- **`vibe-coding-creed-v3.md`** — каноническая «конституция»: 10 заповедей, извлечённый чек-лист (A–H), Definition of Done по уровням T0–T3, приложения (источники, ревью-чеклист, baseline-шаблон CLAUDE.md), план внедрения, глоссарий.
- **`CLAUDE.md.template.md`** — standalone-шаблон, который команды копируют в свои сервисные репо. Семантически дублирует раздел 7.2 креда.

## Семантическая связь файлов

Креду — источник истины. Шаблон — производный артефакт. **При правке любого из двух файлов проверяй второй на согласованность:**

- Раздел 7.2 креда (`vibe-coding-creed-v3.md`, строки ~424–549) = разделы 1–7 шаблона (`CLAUDE.md.template.md`).
- Если меняется заповедь или правило в чек-листе креда — соответствующая строка в шаблоне может стать устаревшей.
- Шаблон явно говорит: «НЕ редактируй разделы 1–7». Корпоративная часть меняется только синхронно с кредо.

## Принципы при редактировании этих файлов

Документ сам себя описывает — соблюдай его правила, когда вносишь правки:

- **Краткость важнее полноты** (раздел 7.1 креда). Цель корневого CLAUDE.md в продуктовом репо — ≤2000 токенов. Если строку можно убрать без потери смысла — убери.
- **Императив, а не описание.** «Используй pytest» — да. «Этот проект исторически использует pytest» — нет.
- **Конкретика, а не идеология.** Правила должны быть проверяемыми.
- Креду структурирован по разделам с номерами (1–9) и подразделами (4.1, 4.2, …). При вставке нового материала сохраняй сквозную нумерацию и не ломай якорные ссылки внутри документа.
- Пиши и отвечай **по-русски** — оба файла и весь дискурс на русском.

## Что НЕ делать

- Не превращай эти документы в README с историей проекта или маркетингом.
- Не дублируй один и тот же текст в обоих файлах вручную — если нужно изменение и там, и там, делай это в одном проходе и оставляй явный комментарий в коммите.
- Не добавляй в шаблон репо-специфику — для неё в шаблоне есть раздел 8 (его заполняет команда-владелец сервисного репо, не этот репо).
- Не вставляй ссылки, которых не было в источниках раздела 5 креда, без явного запроса пользователя — креду намеренно опирается на проверенные публикации.

## Терминология (минимум для навигации)

- **T0 / T1 / T2 / T3** — четыре уровня кода (эксперимент / прототип / внутренний инструмент / продакшен). Каждому соответствует свой Definition of Done в разделе 4 креда.
- **Lethal trifecta** — (приватные данные) + (недоверенный вход) + (внешний канал). Агент не должен иметь все три одновременно. Формулировка Simon Willison.
- **Paved road / Golden Path** — поддерживаемый путь по умолчанию (Spotify, Netflix). Сойти можно через ADR.
- **PRR** — Production Readiness Review (Google SRE), формальный gate T2 → T3.
- **AI-Code Guild** — внутренний орган, владеющий baseline-шаблонами и политикой; пересматривает корпоративную часть раз в квартал.
