# Vibe engineering bible

Корпоративная политика по работе с AI-ассистентами (Claude Code, Cursor, Codex, Claude Cowork). Документационный репо: кода, тестов, сборки здесь нет.

## Содержание

- **[`vibe-engineering-bible.md`](./vibe-engineering-bible.md)** — каноническая «библия». Структура — два завета: **Завет Свободы** (T0, vibe coding) задаёт минимальные правила для личного эксперимента; **Завет Дисциплины** (T1+, vibe engineering) — восемь заповедей, тринадцать грехов (§4½, нормированный шорткодный язык отказа в мердже), DoD по уровням, spec-driven workflow, чек-лист ревьюера AI-кода, план внедрения, роли релиза, глоссарий.
- **[`CLAUDE.md.template.md`](./CLAUDE.md.template.md)** — standalone-шаблон для копирования в продуктовые репо как `CLAUDE.md` или `AGENTS.md`. Единственный источник истины для baseline.

## Как пользоваться

**Команды продуктовых репо:**

1. Скопируйте `CLAUDE.md.template.md` в корень своего репо как `CLAUDE.md` (для Claude Code) и/или `AGENTS.md` (для Codex/Cursor).
2. Заполните только раздел 8 («Репо-специфика»). Разделы 1–7 — корпоративная часть, не редактируйте.
3. Сверьтесь с библией по DoD своей зоны (T1/T2/T3) — §5 библии.

**Контрибьюторы (правка библии или шаблона):**

- При правке любого из двух файлов проверяйте второй на согласованность — маппинг разделов описан в `CLAUDE.md`.
- Принципы редактирования: краткость важнее полноты, императив вместо описания, конкретика вместо идеологии. Подробнее — `CLAUDE.md`.
- Коммиты — Conventional Commits (`feat`, `fix`, `chore`, `docs`).
